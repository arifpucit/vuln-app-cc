Your Mission

This application contains multiple deliberately introduced security vulnerabilities. Your task is to identify each vulnerability in the source code, demonstrate exploitation using appropriate attack vectors and tools, and then patch the code to implement secure defenses against each attack.

Vulnerabilities to Discover

  • SQLi
    SQL Injection

    Manipulate database queries through unsanitized user input to bypass authentication or extract data.

  • XSS
    Stored XSS

    Inject persistent JavaScript saved to the database that executes for every subsequent visitor.

  • XSS
    Reflected XSS

    Inject JavaScript via URL parameters that is reflected in the HTTP response without escaping.

  • Session
    Session Hijacking

    Steal and reuse session cookies to impersonate an authenticated user without knowing their password.

  • Brute
    Brute Force Attack

    Exploit the absence of rate limiting to automatically guess credentials with no lockout mechanism.

  • Crypto
    Weak Password Storage

    Passwords are protected with a broken algorithm, making them trivially reversible if the database leaks.

  • Exposed
    Exposed Database Endpoint

    A misconfigured route allows anyone to download the entire application database over HTTP.

  • CSRF
    Cross-Site Request Forgery

    Forge authenticated requests from a victim's browser to perform actions on their behalf.

How to Approach Each Vulnerability

1
Find

Locate the vulnerable code in the source files and understand the root cause.

2
Exploit

Demonstrate the attack with a working payload, tool, or script to confirm impact.

3
Mitigate

Patch the code using secure coding practices and verify that the fix holds.
